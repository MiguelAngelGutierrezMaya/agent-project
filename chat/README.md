## Description

Chat microservice built with [NestJS](https://github.com/nestjs/nest) framework. This service handles real-time chat functionality with MongoDB as the primary database and AWS SQS for asynchronous message processing.

## Database Configuration

This microservice connects to a single MongoDB cluster with support for multiple databases (multi-tenant architecture).

### Quick Start

1. Copy the environment template:

```bash
$ cp env.example .env
```

2. Configure your MongoDB connection in `.env`:

```bash
MONGODB_URI=mongodb://localhost:27017
MONGODB_PUBLIC_DATABASE=chat_public
```

3. Ensure MongoDB is running:

```bash
# For local MongoDB
$ mongod

# Or using Docker
$ docker run -d -p 27017:27017 --name mongodb mongo:latest
```

### Architecture

- **Public Database**: Contains user mappings and shared information
- **Client Databases**: One database per client, accessed dynamically

### Example Usage

```typescript
import { Injectable } from '@nestjs/common';
import { DatabaseConnectionService } from '@/shared/persistence/mongo/database-connection.service';

@Injectable()
export class MessagesService {
  constructor(private readonly dbService: DatabaseConnectionService) {}

  async getClientMessages(clientId: string) {
    const db = this.dbService.getDatabase(`client_${clientId}`);
    const messages = db.collection('messages');
    return messages.find().toArray();
  }
}
```
